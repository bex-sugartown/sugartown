152-01 Token rename — void → midnight P0
The dark background token is currently named void in some parts of the codebase. This is semantically wrong — midnight is the correct name for this navy-black. Rename the token alias; value stays #0D1226. All components referencing void must be updated to midnight in the same pass.
Acceptance Criteria
  • Token renamed: --st-color-void → --st-color-midnight everywhere it appears in token JSON and CSS
  • Value unchanged: #0D1226
  • Grep for void in all st-* token contexts returns zero hits after rename
  • Dark theme renders identically before and after rename (visual regression check)
152-02 Confirm and name the lime token P0
Lime exists in the dark theme reference (#D1FF1D) but may not have a canonical semantic token name in the design system yet. Confirm whether --st-color-lime or --st-color-accent-lime is the correct name and register it as an official primitive + semantic token. This unblocks EPIC-0150 footer tagline and EPIC-0151 chip variants.
Acceptance Criteria
  • Primitive token defined: --st-color-lime-400: #D1FF1D
  • Semantic alias defined: --st-color-accent-lime: var(--st-color-lime-400)
  • Token appears in token JSON file alongside pink, seafoam, midnight
  • Dark theme epic HTML reference --st-color-lime matches the registered value
  • No hardcoded #D1FF1D hex values remain in any component — all reference the token
152-03 Light mode token set — semantic layer P1
Define the light mode semantic tokens. These are surface, text, border, and accent values for a white-canvas context. Brand colors (pink, seafoam, lime) remain the same in both themes — only canvas, surface, and text roles change. Model the light palette on the Sugartown Pink PRD's Appendix A light mode spec.
Acceptance Criteria
  • Light mode tokens defined under [data-theme="light"] or .theme-light selector (confirm which pattern the codebase uses)
  • Minimum token set covers: canvas bg, surface bg, surface-raised bg, text primary, text secondary/muted, border default, border subtle
  • Light mode values reference existing primitives (gray-100, gray-200, gray-900, etc.) — no new raw hex values without a corresponding primitive
  • Deep Pink Rule respected: --st-color-brand-primary (#FF247D) is NOT used for text on light canvas — --st-color-text-brand (#b91c68) is used instead
  • Token table below is implemented as documented
152-04 Theme switch mechanism — token layer only P1
Wire a minimal theme toggle that swaps between dark and light token sets. Mechanism lives at the HTML or body attribute level (data-theme). No JS framework needed — a simple attribute toggle on <html> is sufficient. Persist to localStorage so preference survives navigation. Test on the existing archive page only.
Acceptance Criteria
  • Theme toggle button renders in the site header (minimal, icon or text — no design required yet)
  • Toggle sets data-theme="light" or data-theme="dark" on <html>
  • CSS uses [data-theme="dark"] for dark tokens and [data-theme="light"] for light — or equivalent pattern confirmed from codebase
  • Preference persists in localStorage across page loads
  • Default theme is dark (Sugartown Dark is the canonical default)
  • Toggle is accessible: keyboard focusable, has ARIA label
152-05 Archive card — basic light mode treatment P1
Apply the new light mode tokens to the existing archive card component. This is not a redesign — it's a token swap. Cards, chips, filter bar, and metadata row should all read from semantic tokens and respond correctly to the theme toggle. This validates the token system is wired end-to-end. Full card audit and new card variants come in EPIC-0151.
Acceptance Criteria
  • Archive card renders correctly in both dark and light themes via the 152-04 toggle
  • No hardcoded hex or RGB values remain in the card component's CSS — all reference --st-* tokens
  • Chip components (st-chip, st-chip--outlined) render legibly on light canvas
  • Filter bar tokens respond to theme change
  • WCAG AA contrast passes for all text in light mode (visual check minimum)
  • Deep Pink Rule applied: pink brand color does not appear as body text on white canvas
