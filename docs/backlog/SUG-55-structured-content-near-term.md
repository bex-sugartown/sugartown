```
Field: readingTime
Type: number (minutes)
Group: metadata
Description: "Estimated reading time in minutes. Manual or computed from word count."
```

### 2. Add `outcomes[]` to case study schema

```
Field: outcomes
Type: array of object
  - metric: string (e.g. "Page load time")
  - value: string (e.g. "2.4s → 0.8s")
  - description: text (optional context)
Group: metadata
```

### 3. Add `series` doc type + wire into article

```
series doc type:
  title → string (required)
  slug → slug
  description → text

article schema additions:
  series → reference to series
  partNumber → number
```

### 4. Add `aiDisclosure` to article + caseStudy schemas

AI Ethics compliance (brief v2026.04.11, principle #13): published articles drafted with AI need a visible disclosure. Currently no schema field or UI rendering exists.

```
Field: aiDisclosure
Type: string
Max: 300 chars
Group: metadata
Description: "AI collaboration disclosure — e.g. 'Drafted with Claude, edited by Bex Head.' Renders below byline on detail pages. Leave blank for fully human-authored content."
```

Render in ArticlePage.jsx and CaseStudyPage.jsx below the byline or in MetadataCard.

### 5. Add `aiGenerated` + `aiTool` to richImage schema

AI Ethics compliance: AI-generated images must carry visible attribution. The existing `credit` field is generic and optional. Adding structured fields enables validation and automated labelling.

```
Field: aiGenerated
Type: boolean
Default: false
Description: "Is this image AI-generated or AI-manipulated?"

Field: aiTool
Type: string
Options: "dall-e" | "midjourney" | "stable-diffusion" | "other"
Hidden: unless aiGenerated === true
Description: "Which AI tool generated this image?"
```

When `aiGenerated` is true, auto-populate `credit` with "Generated by [aiTool]" if credit is empty. Update image naming convention (`docs/conventions/image-naming-convention.md`) to add `ai-` prefix guidance.

### 6. Image alt text audit

Run a Sanity query to identify images with missing or empty alt text across all doc types. The richImage schema has alt text validation, but legacy/migrated images may have slipped through.

```groq
*[defined(sections)] {
  "images": sections[].content[_type == "richImage" && (!defined(alt) || alt == "")]
}[count(images) > 0]
```
